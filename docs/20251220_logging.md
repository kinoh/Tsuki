# Logging decisions (pino)

## Context
- Replace Mastra `ConsoleLogger` with `PinoLogger`.
- Use environment-based behavior instead of `LOCAL`.

## Decisions
- Use `ENV` (fallback `NODE_ENV`) to define runtime environment; default is `development`.
- Enable FCM only when `ENV` is `production`.
- Default log level is `info` unless `LOG_LEVEL` is set.
- Use pino pretty output for non-production; JSON output for production via `overrideDefaultTransports`.
- Replace `console.*` usage in `core/src` with the shared application logger.

## User notes
- Remove `LOCAL` entirely with no compatibility.
- Avoid ambiguous module names like `utils`.
- Do not introduce a logger factory if Mastra is configured in a single location.
- Exclude `core/scripts` and `node_modules` from the console replacement.
